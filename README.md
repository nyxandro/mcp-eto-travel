# MCP ETO Travel

MCP-сервер для поиска туров на `https://eto.travel/search/` через браузерную автоматизацию Playwright.

## Что умеет

- Подключается к сайту поиска туров `eto.travel`
- Ищет подходящий тур по структурированным фильтрам через реальный UI сайта
- Возвращает богатую карточку: название, цену, рейтинг, локацию, описание, картинку и ссылку
- Поддерживает ослабление фильтров, если строгий запрос не дал стабильного результата
- Работает как MCP tool для LLM-клиентов

## Инструменты

- `find_any_tour` — ищет любой подходящий тур по структурированным аргументам (`destination`, `departureCity`, `adults`, `nights`, `month`)

## Архитектура

- `src/index.ts` — точка входа и запуск MCP по `stdio` (стандартный ввод/вывод)
- `src/mcp/server.ts` — регистрация MCP tool `find_any_tour`
- `src/mcp/find-any-tour.ts` — преобразование результата поиска в MCP-ответ
- `src/eto-travel/search-service.ts` — валидация структурированных аргументов и сценарий работы с UI сайта через Playwright
- `src/eto-travel/search-plan.ts` — план fallback-попыток с постепенным ослаблением фильтров
- `src/eto-travel/browser.ts` — адаптер браузера и изоляция Playwright-контракта
- `src/eto-travel/tour-normalizer.ts` — нормализация DOM-результата в стабильный MCP-ответ
- `src/shared/query-parser.ts` — нормализация названий направлений, городов вылета и legacy-парсинг свободного текста для вспомогательных сценариев
- `src/**/tests/*.test.ts` — unit/integration-like тесты для MCP-хендлера, нормализации и search flow

## Как это работает в MCP

- MCP-клиент подключает сервер и получает описание tool `find_any_tour`, его входную схему и инструкцию по заполнению аргументов
- LLM-агент сам извлекает из пользовательской фразы структурированные поля и вызывает tool уже не свободным текстом, а JSON-аргументами
- Сервер валидирует аргументы, нормализует значения под подписи виджета `eto.travel` и выполняет поиск через Playwright
- Если точный набор фильтров не дал пригодной карточки, сервер постепенно ослабляет `month`, `departureCity`, `nights` и явно возвращает это в `relaxedFilters`

## Контракт инструмента

Tool `find_any_tour` принимает такие аргументы:

- `destination` — обязательное направление или страна на русском языке, например `Турция`
- `departureCity` — необязательный город вылета, например `Санкт-Петербург`
- `adults` — необязательное количество взрослых, целое положительное число
- `nights` — необязательное количество ночей, целое положительное число
- `month` — необязательный месяц вылета в нормализованном виде: `январь`, `февраль`, `март`, `апрель`, `май`, `июнь`, `июль`, `август`, `сентябрь`, `октябрь`, `ноябрь`, `декабрь`
- `rawQuery` — необязательная исходная пользовательская фраза для трассировки и отладки

Правила для MCP-клиента и LLM-агента:

- не отправлять один свободный текст вместо структурированных полей
- не выдумывать обязательные значения, если пользователь их не сообщил
- передавать только число взрослых в `adults`, без детей и без текстовых формулировок
- передавать `month` только если пользователь действительно задал месяц явно

Примечание: сервер все еще содержит helper `parseSearchQuery`, но основной MCP-контракт построен вокруг уже структурированных аргументов. Автоматический разбор свободного текста не является главным серверным API.

Пример вызова инструмента:

```json
{
  "tool": "find_any_tour",
  "arguments": {
    "destination": "Турция",
    "departureCity": "Санкт-Петербург",
    "adults": 3,
    "nights": 7,
    "month": "апрель",
    "rawQuery": "Найди тур в Турцию из Санкт-Петербурга на апрель на 7 ночей для 3 взрослых"
  }
}
```

Ожидаемый результат:

```json
{
  "title": "Kemer Star Hotel (Ex. Kemper Dinara Garden) 3*",
  "hotelName": "Kemer Star Hotel (Ex. Kemper Dinara Garden) 3*",
  "price": "87 912 РУБ",
  "dates": "Кемер - центр, Кемер, 500 м до моря",
  "rating": "4.2",
  "location": "Кемер - центр, Кемер, 500 м до моря",
  "description": "Небольшой отель удачно расположен в центре Кемера...",
  "imageUrl": "https://static.tourvisor.ru/hotel_pics/main400/61075.jpg",
```

## Поведение fallback

- Сервис сначала пробует точный набор переданных фильтров
- Если строгий набор фильтров не дает пригодной карточки, сервис постепенно ослабляет `month`, `departureCity`, `nights`
- Ослабленные фильтры явно возвращаются в поле `relaxedFilters`
- Фактически примененные фильтры возвращаются в поле `appliedFilters`

## MCP-инструкции

- Сервер передает модели явную инструкцию: возвращать именно полную карточку отеля, а не только ссылку
- Tool `find_any_tour` принимает структурированные поля, чтобы агент сам извлекал направление, город вылета, месяц, ночи и число взрослых из пользовательской фразы
- Tool `find_any_tour` описан с учетом fallback-логики, чтобы агент понимал, что некоторые не-критичные фильтры могут быть ослаблены
- Это соответствует хорошей практике MCP: у инструмента есть описание, входная схема, выходная схема и поведенческая инструкция

## Важное ограничение

- Сайт `eto.travel/search/` встраивает внешний модуль `tourvisor.ru`. Поэтому надежность поиска зависит от доступности этого виджета и от неизменности DOM-структуры.
- Итоговая ссылка на отель часто ведет не на `eto.travel`, а на `tourcart.ru` — это нормальное поведение виджета.
- При очень жестких фильтрах сервис может ослабить часть условий, чтобы вернуть полезный результат вместо пустой выдачи.
